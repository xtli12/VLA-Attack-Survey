Action Representations as VLA Attack Targets

A robotics-centered table for the survey's action-level attack analysis.

Action Representation Typical Attack Target Closed-Loop Propagation Risk Controller / Recovery Issue Useful Evaluation Signal
Action Tokens Token probabilities, gripper state, pose bins, detokenizer mapping Early token errors may condition later tokens and shift the whole rollout. Controller may clip invalid commands, but semantic wrong-object actions can remain valid. First changed token, targeted token rate, token-to-motion deviation
Action Chunks Multi-step delta poses, primitive windows, static action states Several future control steps can be corrupted before new observations arrive. Replanning frequency and temporal ensemble determine whether drift is corrected or prolonged. Chunk horizon, open-loop duration, persistence after replan
Waypoints Object goal, spatial relation, intermediate pose, affordance map Small semantic shifts can redirect the path to a wrong object or unsafe region. Tracking controller may follow a geometrically valid but semantically wrong waypoint. Endpoint error, wrong-object rate, workspace boundary entry
Trajectories Full path, curvature, velocity profile, attacker-specified endpoint Attack objective can be geometric rather than textual or token-level. Safety filter may prevent collision while still allowing near-boundary or unauthorized motion. Trajectory deviation, time-to-violation, minimum safety margin
Diffusion / Flow Conditioning vector, denoising path, velocity field, sampler, trajectory distribution Smooth distribution shifts may look plausible while biasing actions over time. Sampling and smoothing can attenuate high-frequency attacks but hide low-frequency drift. Distribution shift, multimodality near unsafe regions, post-controller deviation
Skill / API Skill name, tool arguments, permission boundary, precondition check Planner may choose a valid skill for an unauthorized or unsafe goal. Low-level skill may execute correctly even when the high-level authorization is wrong. Unauthorized skill rate, argument-smuggling success, tool audit trace

Use in paper. This table can become a main-text table because it emphasizes robotics mechanisms rather than generic adversarial ML categories.
